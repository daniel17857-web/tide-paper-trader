## 開發順序建議

T-01 → (T-04 + T-02 籌碼軌垂直打通) → T-06 → T-07 → T-08 → T-09 → (T-03 + T-05 技術軌並行補上) → T-10 → T-11。
關鍵路徑上 T-07/T-08 最重,遇三次失敗停損時優先回報,不硬闖。

## 不可踩的線

1. **不寫任何真實下單程式碼**,包括註解掉的樣板、券商 API import。金錢執行永遠由 Daniel 人工。
2. 標的代號/日期一律字串,禁止隱式型別轉換(ETF 管線既有慣例)。
3. 訊號與撮合零隨機性;參數集中 `config.py`,評估期內凍結。
4. 台帳 append-only,禁止改寫歷史。
5. 同K雙觸發以停損計(保守偏差),不得「優化」成對績效有利的規則。
6. 不引入 SPEC/ARCH 未列的第三方套件。

## 風險提示

- **A-01(外匯資料源)**:yfinance 小時K有約 730 天上限與偶發缺棒——T-03 開工先做一次真實抓取煙霧測試,不可只靠 mock。若品質不足即回報,不自行換源。
- **A-02(TAIFEX 端點)**:官網 CSV 格式可能與預期不符——T-02 同樣先煙霧測試真實端點。
- **模擬 ≠ 實盤**:滑價與流動性為固定假設,8 週結果只用於雙軌相對比較,不代表實盤絕對報酬——報告頁需固定顯示此聲明。
- Actions cron 有分鐘級延遲與 DST 偏移(已接受,見 ADR-003/A-05)。

## 回報格式(完成後回 Chat 端驗收)

4. Pages 報告頁網址(含手機截圖)
5. 遺留事項與三次停損觸發紀錄(若有)
