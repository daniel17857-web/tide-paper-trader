# SPEC.md — 潮汐模擬倉(tide-paper-trader)

> **一句話結論**:為 Daniel 建一套「掃描→訊號→計畫→風控→模擬成交→監控」的雙軌模擬交易管線,台指期走籌碼訊號、外匯走純技術訊號,跑 8 週對照兩者期望值,全程不碰真實資金。

## 1. 目標用戶與核心場景

- 用戶:Daniel(單一使用者)
- 核心場景:每日盤後(+外匯關鍵時段)系統自動產出訊號與完整交易計畫,經風控檢核後由模擬撮合引擎虛擬成交,8 週後看雙軌績效對照,決定哪套邏輯值得往實盤半自動推進。
- 成功定義:連續 8 週無人工干預下,管線每日自動產出報告;期末可直接讀出雙軌的勝率/平均R/最大回撤/獲利因子對照表。

## 2. 功能清單與驗收條件

### F-01 TAIFEX 資料 adapter(台指期日頻)
- 描述:每日抓取台指期日K(OHLCV)與三大法人期貨買賣淨部位(外資/投信/自營),累積至本地歷史檔。
- **AC-01**:Given 任一交易日 / When 執行 `fetch_taifex.py` / Then 歷史檔新增該日一列,欄位含 date, open, high, low, close, volume, 外資淨額, 投信淨額, 自營淨額,且 date 無重複。
- **AC-02**:Given 非交易日或資料尚未公布 / When 執行抓取 / Then 程式以 exit code 0 結束並記錄 log「無新資料」,不寫入空列。

### F-02 外匯資料 adapter(日K + 小時K)
- 描述:抓取 EURUSD、USDJPY、GBPUSD、AUDUSD 的日K與小時K,累積至本地歷史檔。
- **AC-03**:Given 網路可用 / When 執行 `fetch_forex.py` / Then 四個貨幣對各自的日K與小時K檔案更新至最新可得K棒,無重複 timestamp。
- **AC-04**:Given 任一貨幣對資料源失敗 / When 執行抓取 / Then 其餘貨幣對照常更新,失敗者記錄至 log,整體 exit code 非 0 以便排程告警。

### F-03 訊號引擎—籌碼軌(台指期)
- 描述:移植既有四訊號至期貨法人資料:淨部位 z-score 異常(≥2σ vs 20 日自身基線)、連續買超 ≥5 日、連續賣超 ≥5 日、外資+投信同向共振。
- **AC-05**:Given 固定的測試資料集(fixture) / When 執行訊號計算 / Then 輸出與 fixture 內預先人工驗算的訊號清單完全一致(逐日逐訊號比對)。
- **AC-06**:Given 歷史資料不足 20 日 / When 執行 / Then z-score 訊號回傳「資料不足」標記而非錯誤或偽訊號。

### F-04 訊號引擎—技術軌(外匯)
- 描述:三種技術訊號:突破(收盤突破 20 日高/低點)、回調(趨勢中拉回至 20EMA 附近後收復)、動能(單根K實體 ≥ 1.5×ATR14 且順勢)。
- **AC-07**:Given 固定 fixture / When 執行訊號計算 / Then 輸出與預驗算清單完全一致。
- **AC-08**:Given 同一貨幣對同日多訊號並發 / When 執行 / Then 依優先序(突破 > 動能 > 回調)只輸出一個進場訊號,其餘記錄為「被抑制」。

### F-05 交易計畫產生器(planner)
- 描述:對每個進場訊號產出:進場區間、停損(結構低/高點與 1.5×ATR14 取較近者)、目標(2R 與前波高/低取較近者)、失效條件、R:R。
- **AC-09**:Given 任一訊號 / When 執行 planner / Then 產出的計畫物件五欄皆非空,且 R:R = (目標-進場)/(進場-停損) 計算正確(誤差 < 0.01)。
- **AC-10**:Given R:R < 1.5 的計畫 / When 執行 / Then 該計畫標記為 rejected,不進入風控與撮合。

### F-06 風控模組
- 描述:依序檢查:部位大小(單筆風險 = 虛擬資金 1%)、總曝險上限(同時持倉 ≤ 3 筆、同軌 ≤ 2 筆)、回撤熔斷(權益自高點回落 ≥ 10% 時停止新倉)。
- **AC-11**:Given 進場價與停損價 / When 計算部位 / Then 部位大小 × 每點價值 × 停損距離 = 虛擬資金 × 1%(誤差 < 1%,台指期以小台每點 50 元、外匯以名目部位換算)。
- **AC-12**:Given 已持倉 3 筆 / When 新計畫進入風控 / Then 回傳 rejected(reason=exposure_cap),不送撮合。
- **AC-13**:Given 權益回撤達 10% 的模擬狀態 / When 新計畫進入風控 / Then 回傳 rejected(reason=drawdown_halt),既有持倉不受影響。

### F-07 模擬撮合引擎 + 交易台帳
- 描述:通過風控的計畫,以「訊號次一根K開盤價 ± 固定滑價」模擬進場;每根新K檢查停損/目標觸發(保守規則:同K同時觸及停損與目標時,以停損計);全部成交寫入台帳。
- **AC-14**:Given 固定 fixture 的一段行情與一筆計畫 / When 逐K推進 / Then 進場價、出場價、出場原因(stop/target/失效)與人工驗算完全一致。
- **AC-15**:Given 任一筆已平倉交易 / When 讀台帳 / Then 該筆記錄含:軌別、標的、訊號類型、進出場時間價位、R 倍數、損益,欄位無缺漏。

### F-08 績效統計與雙軌對照
- 描述:從台帳計算每軌的勝率、平均R、最大回撤、獲利因子,並產出雙軌對照表。
- **AC-16**:Given 內含 10 筆已知結果交易的 fixture 台帳 / When 執行統計 / Then 四項指標與人工驗算一致(誤差 < 0.1%)。

### F-09 排程與報告
- 描述:GitHub Actions 三個排程——16:30 台北(台指期盤後+外匯日K)、16:00 與 21:30 台北(外匯關鍵時段小時K加掃);每次執行後更新 GitHub Pages 靜態報告頁(今日訊號、持倉狀態、雙軌績效)。
- **AC-17**:Given workflow 檔案 / When 以 `act` 或手動 dispatch 執行 / Then 完整管線(抓取→訊號→計畫→風控→撮合→報告)跑通且報告 HTML 產出於 Pages 目錄。
- **AC-18**:Given 報告頁 / When 以手機視窗寬度(390px)開啟 / Then 訊號表與績效表無橫向捲動、可正常閱讀(人工檢核)。

## 3. 非功能需求

| 項目 | 要求 | 驗證方式 |
|------|------|---------|
| 資料完整性 | 標的代號一律字串型別,沿用既有慣例 | pytest 型別斷言 |
| 可重現性 | 同一歷史資料重跑,台帳結果完全一致(無隨機性) | 重跑比對測試 |
| 單次執行時間 | 全管線 < 5 分鐘(GitHub Actions 免費額度考量) | 計時斷言 |
| 語言 | 報告與 log 用台灣正體中文,程式識別字英文 | 人工檢核 |

## 4. 假設(未經用戶確認的推測)

- A-01:外匯資料用 yfinance(免費、小時K約有 730 天上限)——若品質不足,影響 F-02/F-04,屆時再評估付費源。
- A-02:TAIFEX 官網每日盤後 CSV 端點可穩定抓取——若格式變動,影響 F-01。
- A-03:新開獨立 repo,不併入 ETF 管線 repo;共用慣例以文件同步。
- A-04:台指期以小台計價(每點 50 元);外匯以 10 萬名目單位為 1 口基準,不計隔夜利息。
- A-05:「關鍵時段加掃」= 倫敦開盤與紐約開盤各一次,cron 以 UTC 固定,夏令時間偏移 1 小時視為可接受誤差。
- A-06:8 週評估期從管線首次全綠上線日起算,期間不改訊號參數(避免過擬合)。

## 5. 本次不做(Out of Scope)

- 任何真實下單、券商/交易商 API 串接(紅線:金錢執行永遠人工)
- 盤中 tick 級資料與高頻訊號
- 加密貨幣
- 訊號參數最佳化/回測網格搜尋(先拿基準,8 週後再談)
- 議會辯論自動觸發(維持人工啟動)
- 多資金、多帳戶情境
